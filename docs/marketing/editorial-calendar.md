# Editorial Calendar

Use this template to publish one practical builder post every week.

## Weekly cadence

1. Monday: choose one real user pain point
2. Tuesday: collect reproducible commands + screenshots
3. Wednesday: draft post (problem -> commands -> verification -> pitfalls)
4. Thursday: internal review
5. Friday: publish + share in release notes/community

## Post template

```markdown
# [Outcome-focused title]

## Problem
- Who is blocked?
- What failed in the real world?

## Repro
```bash
# minimal commands
```

## Fix pattern
```bash
# stable command sequence
```

## Verification
- What command proves success?
- What artifact proves success?

## Pitfalls
- Common misconfigurations
- Site/provider volatility notes
```
